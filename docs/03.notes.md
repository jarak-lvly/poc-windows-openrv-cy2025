# Notes 

## Version Selection

Several version numbers in `oem/install-openrv-cy2025.ps1` are intentionally hardcoded. The goal of this project is to create a reproducible build environment that matches the Windows development environment used to successfully build OpenRV CY2025.

Where possible, versions were chosen based on one of the following:

- The version documented by the OpenRV project: [Preparing Open RV on Windows](https://aswf-openrv.readthedocs.io/en/latest/build_system/config_windows.html)
- The version installed in the reference Windows development environment.
- The version successfully used to produce a verified OpenRV build.

Hardcoding these values prevents future installer updates from silently changing the build environment.

Examples include:

| Component | Version | Reason |
| ----- | ----- | ----- |
| Windows SDK | 10.0.26100.0 | Installed by the Visual Studio Build Tools workload and used by the successful reference build. Keeping this fixed ensures the include and library paths remain consistent. |
| Python | 3.11.9 | Matches the Python version used by the reference Windows OpenRV build. |
| Qt | 6.5.3 | Required by the current OpenRV Windows build documentation. |
| CMake | 3.31.7 | Matches the reference build environment. |
| Strawberry Perl | 5.42.2.1 | Current working release tested with this build. |
| Rust | 1.97.0 | Current working release tested with this build. |

Not every component is pinned because the OpenRV project requires that exact version. In several cases, the version simply reflects the tested reference environment. Once a build has been validated, reproducibility is generally preferred over automatically consuming newer releases.


## Regenerating the OEM config files (maintainers only)
`openrv-cy2025.vsconfig` and `msys2-packages.txt` in `oem/` were exported from a known-working Visual Studio / MSYS2 installation. You don't need to run this yourself to use the project; it's here for anyone updating these files for a new OpenRV/VFX Platform version.

## On the original windows development vm

The .vsconfig was exported from the Visual Studio 2026 development machine and supplied to the Visual Studio 2022 Build Tools bootstrapper. The installer installed the compatible Build Tools components available in the VS 2022 channel; IDE-specific and otherwise inapplicable entries were not present in the resulting Build Tools installation.

## Export the Visual Studio configuration (example)
**In Powershell:**

Find the installation path:
```
$vswhere = "${env:ProgramFiles(x86)}\Microsoft Visual Studio\Installer\vswhere.exe"

$vsInstallPath = & $vswhere `
    -latest `
    -products * `
    -property installationPath

$vsInstallPath
```

returns path that should be used for `--installPath <path>` during export:
```
C:\Program Files\Microsoft Visual Studio\18\Community
```

**Then export the configuration:**
```
& "${env:ProgramFiles(x86)}\Microsoft Visual Studio\Installer\setup.exe" `
    export `
    --installPath $vsInstallPath `
    --config "C:\temp\openrv-cy2025.vsconfig"
```

### Export the MSYS2 packages:
**In Powershell:**
```
C:\msys64\usr\bin\bash.exe -lc "pacman -Qeq > /c/temp/msys2-packages.txt"
```
